Cara membuat VM dengan virt-manager

Dengan menggunakan virt-manager kita bisa lebih mudah dalam membuat sebuah VM karena tampilan UI-nya mudah dipahami. Kita bisa mengatur alokasi disk, memilih OS, mengelola virtual hardware, dan mengatur penyesuaian lainnya sebelum membuat VM. Ada 4 cara yang disediakan virt-manager dan dapat dipilih dalam membuat VM. Pada blog kali ini saya akan mengajak kalian untuk membuat VM dengan 4 cara yang disediakan oleh virt-manager. Pastikan host/laptop/PC kalian sudah terinstal package KVM dan virt-manager. Cek juga perangkat yang kalian gunakan sudah kompatibel untuk divirtualisasi. Gasss, kita mulai saja!

Cara Pertama: Local install media (ISO image or CDROM)

Untuk cara yang pertama ini kita memerlukan CD-ROM, DVD, atau ISO image dari OS yang akan diinstal pada VM. Langkah-langkahnya adalah sebagai berikut:

  1. Buka desktop atau aplikasi virt-manager
  2. Klik tombol ikon monitor kecil pada pojok kiri atas di bawah tulisan File
  3. Pilih opsi pertama pada pilihan yang disediakan, selanjutnya klik Forward
  4. Setelah itu kita bisa memilih dan memasukan ISO image dari OS yang akan diinstal pada VM. Pastikan sudah mendownload ISO image dari OS yang diinginkan. Disini saya menggunakan ISO image dari OS Ubuntu 18. Jika sudah, lanjut klik Forward. Ketika file ISO image sudah dimasukan ke dalam virt-manager, biasanya virt-manager akan mendeteksi OS dari file ISO image secara otomatis. Jadi kita bisa mengetahui, bahwa kita sudah memasukan file ISO image dari OS yang sesuai dengan keinginan kita.
  5. Kita juga bisa mengecek apakah file ISO yang sudah didownload itu benar-benar OS yang dipilih.

    Caranya dengan menjalankan perintah berikut:

    $ osinfo-detect ubuntu-18.04.6-desktop-amd64.iso

    Output dari perintah di atas:

    Media is bootable. Media is an installer for OS 'Ubuntu 18.04 LTS (x86_64)' Media is live media for OS 'Ubuntu 18.04 LTS (x86_64)'

    Atau bisa juga dengan perintah di bawah ini:

    $ file ubuntu-18.04.6-desktop-amd64.iso

    Output dari perintah di atas:

    ubuntu-18.04.6-desktop-amd64.iso: ISO 9660 CD-ROM filesystem data (DOS/MBR boot sector) 'Ubuntu 18.04.6 LTS amd64' (bootable)
  6. Kemudian kita bisa menentukan jumlah memory, CPU, dan juga disk untuk VM yang akan dibuat. Atau kalian juga dapat membuat kustomisasi disk sendiri. Di sini saya memilih konfigurasi yang mudah saja, mungkin di blog selanjutnya saya akan membahas tentang kustomisasi disk pada virt-manager. Jika sudah menentukan jumlah memory, CPU, dan disk. Lanjut dengan klik Forward.
  7. Selanjutnya kita bisa memberi nama pada VM yang akan dibuat. Disini juga kita diberi preview spesifikasi dari VM yang akan dibuat. Klik Finish untuk memulai installasi.
  8. Konfigurasi installasi VM sesuai keinginan kalian. proses installasi VM sekitar 5-10 menit. Jika sudah akan muncul tampilan dari VM yang sedang berjalan pada virt-manager. Pembuatan VM dengan cara pertama sudah selesai.

Cara Kedua: Network Install (HTTP, HTTPS, or FTP)

Pada cara ini, kita hanya memerlukan URL dari OS yang diinginkan untuk membuat VM. Di dalam URL tersebut, sudah disediakan file untuk installasi OS dan paket-paket yang yang dibutuhkan. Contohnya disini saya akan menginstall VM dengan OS Rocky Linux menggunakan cara ini.

URL install Rocky Linux:
https://dl.rockylinux.org/vault/rocky/9.3/BaseOS/x86_64/os/

  1. Untuk membuat VM dengan cara ini, pilih opsi kedua dari pilihan yang disediakan, selanjutnya klik Forward
  2. Tambahkan URL dari OS yang diinginkan, contohnya saya menggunakan Rocky Linux. Lanjut klik Forward
  3. Kemudian atur jumlah memory, CPU, disk, dan nama untuk VM yang akan dibuat. Jika sudah klik Finish
  4. Tunggu proses installasinya sekitar 5-10 menit
  5. Setelah proses installasi selesai, akan muncul tampilan pengaturan seperti di bawah ini. Teman-teman bisa mengikuti instruksi/panduan yang disediakan Rocky Linux
  6. Dan VM dengan distro Rocky Linux sudah berjalan
  7. Berikut referensi beberapa URL untuk membuat VM dengan distro OS lainnya:

    - URL centos-9-stream

    - URL debian

    - URL opensuse

    - URL archlinux

Cara Ketiga: Import existing disk image

Untuk cara ini, kita bisa membuat VM baru dengan mengimport disk yang sudah ada. Baik itu dari disk yang sudah digunakan untuk membuat VM atau disk baru tetapi belum digunakan sama sekali. Dengan kata lain, kita bisa memanfaatkan disk yang sudah kita miliki untuk membuat VM baru. Kita juga bisa mengustomisasi disk yang sudah ada dengan menginstall paket, membuat file, menambahkan user, dll. Berikut Langkah-langkah pembuatan VM dengan cara ketiga:

  1. Langkah pertama, jika teman-teman sudah memiliki disk atau cloud image dari pembuatan VM sebelumnya, dengan cara ketiga ini kita bisa menggunakan disknya lagi. Disini saya akan memberi contoh dengan menggunakan disk image baru dari OS Ubuntu Bionic, lalu saya akan membuat salinan dari disk tersebut. Tidak lupa juga, saya akan melakukan sedikit kustomisasi pada salinan disk Ubuntu Bionic dengan menambahkan beberapa konfigurasi.

    - Membuat salinan disk image, caranya seperti berikut:

    $ sudo qemu-img create -b /var/lib/libvirt/images/bionic-server-cloudimg-amd64.img -f qcow2 -F qcow2 /var/lib/libvirt/images/ubuntu18.img 15G ###Output dari perintah diatas### Formatting 'ubuntu18.img', fmt=qcow2 cluster_size=65536 extended_l2=off compression_type=zlib size=16106127360 backing_file=bionic-server-cloudimg-amd64.img backing_fmt=qcow2 lazy_refcounts=off refcount_bits=16 ###Cek informasi dari disk tersebut### $ file /var/lib/libvirt/images/ubuntu18.img ubuntu18.img: QEMU QCOW2 Image (v3), has backing file (path bionic-server-cloudimg-amd64.img), 16106127360 bytes

    - Untuk kustomisasi disk, saya menggunakan tools virt-customize. Berikut syntax dari perintahnya:

    $ sudo virt-customize -a /var/lib/libvirt/images/ubuntu18.img --update --run custom-script.sh ###Output dari perintah diatas### [ 0.0] Examining the guest ... [ 4.5] Setting a random seed virt-customize: warning: random seed could not be set for this type of guest [ 4.6] Setting the machine ID in /etc/machine-id [ 4.6] Updating packages [ 46.7] Running: custom-script.sh [ 61.2] Finishing off

    - Kustomisasi disk yang saya tambahkan antara lain seperti membuatkan user, menginstall package, dll. Instruksi kustom saya tuliskan di dalam script dengan nama custom-script.sh, file yang berisi script kustom tersebut akan dieksekusi setelah tools virt-customize melakukan update packages. Berikut isi dari file custom-script.sh:

    #!/bin/bash ## Create and add user mkdir -p /home/zero/.ssh useradd zero -d /home/zero -s /bin/bash echo "zero:zero123"|chpasswd echo "zero ALL=(ALL) NOPASSWD:ALL" | tee /etc/sudoers.d/zero echo "root:root123"|chpasswd ## Installing package apt-get install apache2 -y apt-get install openssh-server -y ## Configure service apache2 mkdir -p /var/www/html/ echo "knock knock ubuntu!" > /var/www/html/index.html chmod 777 /var/www/html/index.html
  2. Langkah kedua, setelah menyiapkan disk, lanjut pada proses pembuatan VM. Pada menu virt-manager pilih opsi ketiga dari pilihan yang disediakan. Selanjutnya klik Forward
  3. Ketiga, tambahkan disk image yang sudah dimodifikasi tadi, lalu pilih OS dari disk image tersebut. Jika sudah klik Forward
  4. Kemudian atur jumlah memory, CPU, disk, dan nama untuk VM yang akan dibuat. Jika sudah klik Finish
  5. Tunggu proses installasinya sekitar 1-2 menit
  6. Dan akhirnya VM pun sudah jadi, kalian bisa melakukan pengecekan apakah konfigurasi atau kustom yang sudah diterapkan ke dalam VM itu terinstall atau tidak. Di sini saya akan mencoba melakukan pengecekan dengan melakukan login ke dalam VM dengan user yang sudah ditentukan di dalam script dan lanjut cek package apache2

Cara Keempat: Manual install

Dengan memilih cara keempat yaitu Manual install, kita dapat membuat VM terlebih dahulu meskipun belum memiliki media installasi seperti ISO image atau media lainnya. Perlu diingat juga sebelum melakukan booting ke VM, kita harus sudah menambahkan ISO agar bisa melakukan installasi. Lebih lanjutnya kalian bisa mengikuti langkah-langkah berikut ini:

  1. Langkah pertama, pilih opsi keempat atau terakhir dari menu virt-manager. Selanjutnya klik Forward
  2. Langkah kedua, pilih OS yang akan diinstall(Ini hanya memilih OS saja, langkah ini tidak akan melakukan penginstallan ISO OS). Disini saya memilih Ubuntu 18
  3. Kemudian atur jumlah memory, CPU, disk, dan nama untuk VM yang akan dibuat. Jika sudah klik Finish Catatan, dibagian seperti gambar di bawah ini, checklist menu Customize configuration before install
  4. Setelah itu, kalian akan melihat menu seperti gambar di bawah ini. Langkah selanjutnya, kita akan menambahkan disk yang berisi file ISO dari OS yang sudah ditentukan. Untuk caranya pertama klik tombol Add Hardware di sebelah kiri bawah.
  5. Kemudian, pilih menu Storage dan checklist di bagian Select or create custom storage, lalu klik Manage
  6. Pilih file ISO untuk dijadikan disk dan juga agar VM bisa melakukan booting, selanjutnya klik Choose Volume
  7. Pada bagian Device type, pilih CDROM device, lalu klik Finish
  8. Masuk pada bagian Boot Options, checklist SATA CDROM agar VM dapat melakukan booting ke OS Ubuntu 18, terus klik Apply
  9. Lanjut klik tombol Begin Installation pada bagian kiri atas dari gambar di bawah ini
  10. Setelah itu lakukan konfigurasi untuk mengatur setingan VM sesuai keinginnan kalian, jika sudah tunggu proses installasi VM sekitar 1-5 menit
  11. Jika installasi VM sudah selesai biasanya terdapat tombol untuk melakukan reboot. Tekan tombol reboot pada VM, selanjutnya kita bisa menghentikan operasi VM atau shutdown terlebih dahulu. Karena kita perlu menghapus disk ISO yang sudah digunakan menjadi bootable, agar kita bisa mengakses atau login ke dalam VM. Jika disk ISO tidak dihapus, maka akan terjadi looping dan kembali mengulangi proses installasi VM. Kita bisa menghentikan operasi VM dengan cara menekan tombol shutdown atau Force Off(secara paksa).
  12. Kemudian, masuk ke menu detail pada VM, pilih menu SATA CDROM 1, lalu klik tombol Remove pada bagian kanan bawah. Jika kalian ingin sekalian menghapus file ISO pada laptop, kalian bisa checklist menu Delete associated storage files. Disini saya hanya ingin menghapus disk ISOnya saja. Jadi saya tidak menchecklist menu tersebut. Setelah itu klik Delete
  13. Pastikan pada menu Boot Options, opsi disk yang tadi tidak dicheklist, menjadi terchecklist. Fungsinya agar kita bisa booting ke dalam VM yang sudah dibuat, lanjut klik Apply
  14. Klik tombol Play/Power On/Segitiga pada bagian atas. Untuk menghidupkan VM
  15. Tunggu proses booting. Jika sudah muncul tampilan seperti gambar di bawah ini, kalian bisa login dengan user yang sudah dibuat pada saat proses installasi VM. Disini saya login dengan kredensial sebagai berikut username: manual password: man123 *ini hanya untuk contoh jangan sesekali menggunakan kombinasi password seperti yang di atas.
  16. Dan akhirnya VM pun sudah bisa berjalan
